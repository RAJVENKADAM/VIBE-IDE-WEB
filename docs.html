What is VIBE?

VIBE is a Visual Intent Build Engine, designed to convert simple Human readable text into UI elements with original raw codes (html, css, js, py). It bridges the gap between intent and code by using human-readable syntax.

How to Download

Guidelines

1. Download the executable installer using the button below.
2. Run the setup and grant necessary permissions.
3. Launch the VIBE-IDE from your desktop shortcut.

Download VIBE-IDE

Explore

Explore the features and interface of VIBE-IDE to maximize your productivity.

1. Open Folder

Click the "Open Folder" button and select the directory where you want to save your project. This ensures the compiler has the correct path to generate your HTML, CSS, and JS files.

2. Where to Code

Navigate to the VIBE folder in the explorer. Click on app.vibe to begin. Write your code here; the IDE automatically converts it into HTML, CSS, JS, or Python.

3. Inspection

Click the Inspection tab in the IDE at center panel. This allows you to view the live generated code. You can see the HTML, CSS, JS, or Python outputs by clicking the corresponding files in the explorer.

4. AST (Abstract Syntax Tree)

Curious about how your code is processed? The AST tab displays the structure of your VIBE code as it is tokenized and transformed into the original output code.

5. Switch Language

VIBE is multi-target. Use the Switch Language button to change the output format. The compiler will instantly re-generate your project into the selected language.

6. Live Preview

The Preview panel allows you to see your UI results lively. Every change you make in app.vibe reflects here in real-time.

7. Using Original Code for Production

VIBE-IDE isn't just a playground; it generates professional, production-ready source code. Once you've built your "Vibe," follow these steps to take your project live.

Step 1: Locate the APP Folder

Navigate to your project directory. All converted raw source files (HTML, CSS, JS, and Python) are automatically organized and stored inside the APP folder.

Step 2: Open in External Editor

Right-click the APP folder and select "Open with Code" (or your preferred editor like VS Code or PyCharm). Here, you will see the clean, human-readable code generated by the VIBE Engine.

Step 3: Version Control & Deployment

Push your APP folder to GitHub or any VCS. From there, you can deploy to platforms like Netlify, Vercel, or Heroku just as you would with any traditional project.

Step 4: Production Ready

Your high-level VIBE intent has now been successfully compiled into original raw code, optimized and ready for a live production environment.

Core Rules

Follow these structural rules to ensure the VIBE compiler processes your intent correctly.

1. The Dot (.)

Every command acts like a sentence and must end with a period. This tells the engine the instruction is complete.

✔ Correct:
create text "Hello".
✘ Incorrect:
create text "Hello"

2. Double Quotes (" ")

All content (labels) and style values (sizes, colors) must be wrapped in double quotes.

create button "Submit" with bg "green".

3. Styling Keyword

To add styles, you must use the with keyword after the element content.

create box "Header" with bg "black" color "white".

4. Case Insensitivity

VIBE doesn't care if you shout or whisper. Commands can be any case.

// These are identical:
CREATE TEXT "Title".
create text "Title".

5. One Line, One Element

Each individual UI component must occupy its own line in the app.vibe file.

create text "Line 1".
create text "Line 2".

Elements

Text

Displays a text element on the screen.

create text "Hello World" with color "red" size "24px".

Button

Creates a clickable button with specific styles.

create button "Click Me" with bg "blue" color "white" padding "10px".

Box

A container used to group other elements.

create box "Card Area" with bg "#111" color "white" padding "20px" radius "10px".

Link

A hyperlink to an external or internal URL.

create link "Google" with url "https://google.com" color "blue".

Styling Properties

Styles are applied using key-value pairs following the with keyword. Multiple properties can be chained in a single line.

Property Action Example Usage
bg Sets the background color (Name or Hex). (action + element) with bg "blue"
color Sets the text color. (action + element) with color "white"
size Adjusts the size of the text. (action + element) with size "20px"
padding Adds internal spacing inside the element. (action + element) with padding "15px"
radius Rounds the corners of the element. (action + element) with radius "10px"
width Sets the horizontal width. (action + element) with width "300px"
shadow Adds a drop shadow for depth. (action + element) with shadow "0 4px 8px rgba(0,0,0,0.1)"

Combined Example

create box "Full Style" with bg "purple" color "white" padding "20px" radius "12px" shadow "0 10px 20px rgba(0,0,0,0.2)".
